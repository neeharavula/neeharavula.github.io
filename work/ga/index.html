Overview

General Atomics ↗  is a global defense company with several affiliate divisions including Aeronautical Systems (GA-ASI) and Electromagnetic Systems (GA-EMS). I spent the summer working as a Software Developer Intern on a Modeling and Simulation team under GA-ASI's Autonomy and AI department, which is responsible for the delivery of military-grade unmanned aerial vehicle (UAV) ground station flight simulation software.

Task

My team utilized an application called AVSim, which simulates the flight of UAVs including real-time aircraft telemetry data. AVSim can communicate the simulated telemetry data via UDP messaging to other applications such as a 3D virtual world simulator app or to a physical ground station computer itself to see the aircraft in flight.


One fallback of AVSim, however, was that it did not offer a way to visually see the data it was sending out to other apps/computers. Therefore, my task was to develop a monitor application that could be used in conjunction with AVSim to allow developers or ground station pilots to easily examine the telemetry messages being generated by AVSim. This would help users quickly identify errors in the data that might be affecting the behavior of the aircraft during flight and ensure that the right data is being passed to the apps/ground station as needed.

Process

To accomplish this task, I worked through the Software Development Life Cycle (SDLC) which involved the following:

  • Establishing technical and functional requirements
  • Identifying potential use cases and error-handling based on Interface Control Documentation (ICDs)
  • Creating wireframes/mockups of the monitor graphical user interface (GUI) and making modifications according to feedback from design reviews until achieving a final design that satisfied the established requirements and use cases
  • Performing iterative development in Python, beginning back-end with the implementation of a multi-threaded UDP client-server program, and then moving towards front-end with the construction of a graphical user interface (GUI) using Python's Tkinter toolkit
  • Testing with my own test client (which contained a simple, parsable telemetry message that mimicked an AVSim message) and with AVSim itself, and making modifications as needed until the expected behavior was achieved

    • Reflections

    During this internship I encountered a lot of 'firsts' that helped me grow immensely in both the technical and professional domains. Coming from primarily back-end experience, this was my first time working full-stack and I found that I really enjoy the front-end and UI/UX design phases of the SDLC along with the back-end development phase. It was also my first time implementing a multi-threaded client-server program. While intimidating at first, I was able to achieve the desired functionality by using an iterative/incremental approach to development:

  • I first started with a basic client and server that could communicate over a UDP sockets
  • Then in my client module, I implemented a message class object, where class members represented aircraft telemetry data fields (ex: altitude, pitch, etc.) similar to the structure of AVSim's messages, and ensured that my server was able to receive and parse the message data as required
  • Finally, I added threading functionality to my server in order to be able to process and display multiple message types from AVSim on my GUI at a time

    • This was also my first time working within a collaborative scrum environment, where I learned how to break down my project into more manageable chunks, communicate my progress and issues effectively, and work through sprint cycles efficiently. Overall, it was an incredibly rewarding experience that allowed me to apply object-oriented design and systems programming concepts I learned in school to building a real-world application, and helped me gain more confidence as an aspiring software developer!
