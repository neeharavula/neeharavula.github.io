Overview

Northwestern Mutual (NM) ↗  is a financial services and life insurance company based in Milwaukee, WI. I spent the summer working on a client communications team within NM's Engineering Solutions Delivery division, which is responsible for modernizing the development and delivery of communications and applications used by over 10,000 field representatives. These client communications can consist of physical contracts, statements, bills, and correspondence, as well as digital notifications sent via text or email.

Contributions

To create and deliver a new letter or communication, my team worked through a letter onboarding process with the following steps:

  • Gather the client data (policy account numbers, address, billing info, etc.) needed to create the specific communication
  • Check if the communication must be sent digitally or physically via mail
  • Check if there are multiple pages/documents for the communication, and if so, bundle them together
  • Store communication bundle(s) in digital or physical storage
  • Deliver the communication based on the client's preferences


    • Task 1: RMD Notification (Pre-Existing Communication)

    I had the opportunity to get involved with this process with the development of an existing email notification for Required Minimum Distribution (RMD), a communication that specifies the minimum amounts that US tax law requires someone to withdraw annually from a traditional IRA or employee-based retirement account. This is an example of a common type of communication that gets sent out to field representatives about their clients and the policies they signed up for.

    I was responsible for modifying the JavaScript code to update the data and structure of this notification according to new product requirements (ex: ensure that the client information table in the email is populated with the correct data, produce an excel attachment for the table data if it exceeded a certain length, etc.). I also performed end-to-end testing using Postman API and AWS DynamoDB to ensure that the email notification was successfully able to pass through each step of the aforementioned letter onboarding process and updated as expected upon final delivery.


    Task 2: Prospect Registration Letter (Brand New Communication)

    Towards the end of my internship, I also had the opportunity to work on creating and onboarding a brand new marketing communication, a Prospect Registration Letter, designed for prospective clients who don't yet own any NM products or policies. This involved creating a new client communication definition, which serves as the recipe containing the steps needed to create this document, determine what data is needed for it, determine how it gets delivered, etc. before it is submitted to the letter onboarding process.


    Task 3: Software Vulnerability Remediation

    I was also able to engage in software vulnerability management. Vulnerabilities are potential security exposures in a piece of software or application that can cause issues if left unaddressed. I was involved in the process of identifying and remediating 12 vulnerabilities that were affecting some of the code in our department using Grafana's Vulnerability Dashboard, before those code changes were deployed to production.

    Reflections

    Throughout the summer I had the opportunity to work with software and applications I wasn't familiar with before, including MySQL Workbench, Postman, and AWS DynamoDB, and was able to work with and learn from professionals from teams both inside and outside my department which really helped me expand my technical skills and open my eyes to all that the fintech world has to offer. Overall, my summer at NM was an incredible experience— I learned so much working at the intersection of finance and technology, met so many amazing people, and had a great time exploring Milwaukee!
